# Git の使い方 by Gemini
Gitへようこそ！最初は難しく感じるかもしれませんが、Gitは「ファイルのバックアップを賢く取り、チームで共有するためのツール」です。

VS Codeを使えば、難しいコマンドを覚えなくてもボタン操作で進めることができます。まずは基本から押さえていきましょう。

---

## 1. Gitの基本：イメージで理解する

Gitは、ファイルの「変更履歴」を記録するツールです。

* **リポジトリ (Repository):** プロジェクトの全データと履歴が保管される「貯蔵庫」です。
    * **リモートリポジトリ:** ネット上（GitHubなど）にある、チーム共有の貯蔵庫。
    * **ローカルリポジトリ:** 自分のPC内にある、自分用の貯蔵庫。
* **コミット (Commit):** ファイルの変更を「セーブ」すること。誰が・いつ・何をしたかというメッセージを添えます。
* **プッシュ (Push):** 自分のPCでの「セーブ（コミット）」を、共有の貯蔵庫（リモート）へアップロードすること。
* **プル (Pull):** 他の人がアップロードした最新の変更を、自分のPCへダウンロードすること。

---

## 2. VS CodeでのGit操作（GUI編）

VS Codeを使えば、画面左側の「ソース管理」アイコン（枝分かれしたようなマーク）から直感的に操作できます。

### ① プロジェクトの準備（クローン）
まずは、共有されているリポジトリを自分のPCにコピーします。
1. VS Codeを開き、`Ctrl + Shift + P` (Macは `Cmd + Shift + P`) を押します。
2. 「Git: Clone」と入力して選択します。
3. リポジトリのURLを貼り付け、保存先を選びます。

### ② 変更をセーブする（コミット）
ファイルを編集したら、その変更を記録します。
1. 左側の**「ソース管理」アイコン**をクリックします。
2. 変更したファイル名の横にある **「+」ボタン**（ステージング）を押します。
3. 上部の入力欄に**「変更内容のメモ（コミットメッセージ）」**を書きます。
4. **「コミット」ボタン**を押します。
   * *注意：これだけでは、まだチームには共有されていません！*

### ③ チームに共有する（プッシュ）
1. 画面下部にある「同期（矢印のマーク）」または「変更の同期」ボタンを押します。
2. これで、あなたのコミットがリモートリポジトリに反映されます。

### ④ 最新の状態を取り込む（プル）
作業を始める前には、必ず他の人の変更を取り込みます。
1. 「ソース管理」メニューの「…（もっと見る）」から **「プル」** を選択するか、同期ボタンを押します。

---

## 3. チームで使う際の注意事項（これだけは守ろう！）

チーム開発をスムーズにするための、3つの鉄則です。

1.  **作業前に必ず「プル」する**
    * 古い状態のまま作業を始めると、他の人の変更と衝突（コンフリクト）しやすくなります。朝一番や、新しい作業を始める時は必ずプルしましょう。
2.  **コミットメッセージは具体的に**
    * 「修正」だけではなく、「〇〇画面のボタンの色を変更」のように、後で誰が見てもわかるように書きましょう。
3.  **衝突（コンフリクト）を恐れない**
    * 同じ箇所を同時に編集すると「競合」が発生します。VS Codeが「どちらの変更を残しますか？」と聞いてくれるので、慌てずチームメンバーに相談して解決しましょう。

---

最初は「まずプルして、作業して、コミットして、プッシュする」という流れを繰り返すだけで大丈夫です。わからないことがあれば、すぐにチームの詳しい人に画面を見せて相談するのが上達の近道ですよ！

---

## 4. ブランチとは：「パラレルワールド」を作る機能
Gitの「ブランチ（Branch）」は、チーム共同で作業する上で最も重要な機能です。これがあるおかげで、他の人の作業を邪魔することなく、自分の作業を進めることができます。

ブランチ（枝）とは、プロジェクトの履歴を枝分かれさせる機能です。

* **mainブランチ（本番）:** 常に「正しく動く最新の状態」を保つメインの道です。
* **フィーチャーブランチ（作業用）:** 自分の担当機能を作るために、メインの道から一時的に作った「脇道」です。



例えば、「Aさんがトップ画面を修正している間、Bさんがお問い合わせフォームを作っている」という状況でも、お互いの作業中のコードが混ざり合ってバグを引き起こすことがありません。

---

## 5. VS Codeでの操作方法（GUI編）

VS Codeの左下やソース管理画面から、簡単にブランチを切り替えられます。

### ① 新しいブランチを作る
1. VS Codeの左下にある「main（または現在のブランチ名）」をクリックします。
2. 画面上部に出るメニューから **「新しいブランチの作成...」** を選びます。
3. ブランチ名（例：`feature-login-screen`）を入力してEnter。
   * これで、あなたは「脇道」に入りました。ここでの変更は`main`には影響しません。

### ② ブランチを切り替える（チェックアウト）
1. 左下のブランチ名をクリックします。
2. 切り替えたいブランチ名を選択します。
   * 作業を中断して他の人の手伝いをする時などに使います。

### ③ 自分のブランチを共有する（プッシュ）
1. 自分のブランチでコミットしたら、右上の「ブランチの公開」または雲のマーク（プッシュ）を押します。
   * これで、GitHubなどのリモートリポジトリに自分の「脇道」がアップロードされます。

---

## 6. チーム開発での「ブランチ運用」ルール

チームでGitを使う際は、一般的に以下のような流れ（ワークフロー）で進めます。

1.  **mainから枝分かれする:**
    必ず最新の`main`から新しいブランチを作ります。
2.  **ブランチ名にルールを決める:**
    「誰が・何をしているか」わかる名前にしましょう。
    * 例：`user/suzuki/add-contact-form`
3.  **直接mainにプッシュしない:**
    `main`は「完成品」を置く場所です。直接書き込むのではなく、必ず自分のブランチで作業します。



---

## 7. 「プルリクエスト（PR）」で合流する

自分の作業が終わったら、脇道（ブランチ）からメインの道（main）に合流させます。この時、**「プルリクエスト（Pull Request）」**という仕組みを使います。

1.  **依頼する:** 自分のブランチをプッシュした後、GitHubなどの画面上で「私の変更をmainに合流させてください」と依頼を出します。
2.  **レビューを受ける:** チームメイトにコードを見てもらい、「OK」をもらいます。
3.  **マージ（合流）:** 最後にボタンを押して`main`に合流させます。

> **ポイント：なぜこんな面倒なことをするの？**
> 誰かが間違えてプログラムを壊しても、`main`に合流する前に他の人が気づけるようにするためです。これがチームの品質を守る砦になります。

## 8. 付録：CUI操作（コマンドライン）

| 操作 | コマンド |
| :--- | :--- |
| **複製** | `git clone [URL]` | リポジトリをコピーする |
| **確認** | `git status` | 現在の変更状態を確認する |
| **追加** | `git add [ファイル名]` | 変更を記録対象（ステージ）にする |
| **記録** | `git commit -m "メッセージ"` | メッセージを付けてセーブする |
| **送信** | `git push origin [ブランチ名]` | リモートにアップロードする |
| **受信** | `git pull origin [ブランチ名]` | リモートから最新を取り込む |
| **作成と切り替え** | `git checkout -b [新しいブランチ名]` |
| **一覧表示** | `git branch` |
| **切り替え** | `git checkout [既存のブランチ名]` |
| **合流（マージ）** | `git merge [ブランチ名]` (現在いるブランチに合流させる) |
